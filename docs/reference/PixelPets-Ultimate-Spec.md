# PixelPets 终极设计白皮书 (Ultimate Design Spec)

**版本**：v3.0 (Final)
**日期**：2026-04-28
**状态**：正式定稿

## 一、项目愿景 (Mission)

PixelPets 旨在将枯燥的 AI 订阅额度消耗转化为具有生命力的像素陪伴体验。它不仅是一个额度监控工具，更是一个寄生在 macOS 菜单栏的、受 AI 驱动的"数字生物"生态系统。

通过**"瞬时反馈"**与**"长期养成"**的双轨机制，提升开发者的情绪价值，并以模块化架构确保宠物生态的无限扩展性。

## 二、视觉与技术标准 (Visual & Technical Standards)

### 1. 16×16 绝对物理网格

- 菜单栏图标必须在 **16×16 像素**内完成设计。
- **禁用抗锯齿**：使用 `image-rendering: pixelated` 和 `shape-rendering: crispEdges`。
- **绝对黑边**：1px 纯黑 (`#000`) 轮廓线，通过**移除四角像素**实现视觉上的圆角（Kawaii Look）。
- Popover 面板大图：64×64 px（高清采样，仍保持像素感）。

### 2. 分层渲染架构 (Layered Rendering)

每个宠物由以下 6 个图层叠加而成，顺序不可变更：

| 层级 | 名称 | 内容 | 动态性 |
|:---|:---|:---|:---|
| 1 | **Skin** | 基础色 / Agent 渐变 | 随 Agent 切换 |
| 2 | **Shading** | 高光（左上 1px 白色，opacity 0.2）+ 暗部（右下 1px 黑色，opacity 0.2） | 静态 |
| 3 | **Body** | 黑色轮廓与结构线（去四角圆角） | 静态 |
| 4 | **Face** | 屏幕 / 眼睛 / 表情（12 种状态） | 随 Hook 切换 |
| 5 | **Accessories** | 配件插槽（耳机、天线、披风等） | 随等级解锁 |
| 6 | **FX** | 环境特效（Zzz、烟花、扫描线、拖影） | 随状态触发 |

> **Swift 实现建议**：使用 `Canvas` + `TimelineView` 逐层绘制，确保 Retina 屏下 `nearestNeighbor` 采样，避免像素模糊。

### 3. Agent 配色映射 (Color Key)

| Agent | 定稿颜色 | 类型 | 备注 |
|:---|:---|:---|:---|
| **Claude** | `#D97757` | 纯色 | Terracotta 橙，白色屏幕 |
| **Gemini** | `conic-gradient(#EA4335, #FBBC05, #34A853, #4285F4)` | 圆锥四色渐变 | 模拟星芒旋转感，方向顺时针 |
| **Codex** | `linear-gradient(180deg, #7851FB, #2853FF)` | 垂直蓝紫渐变 | 基于 Codex App 图标取色，见截图 |
| **OpenCode** | `#000000` | 纯色 | 黑身白屏，高对比黑白风格 |

> **Codex 取色说明**：根据官方 App 图标截图，Codex 为蓝紫渐变，非纯蓝。顶部偏紫 (`#7851FB`)，底部偏蓝 (`#2853FF`)。V5 旧版 `#1A73E8` 已废弃。

## 三、五大核心种族 (MVP)

MVP 阶段支持以下五类具有截然不同物理质感和动画逻辑的种族。所有种族均遵循 `PetCore` 框架，通过替换 `FaceProvider` 和 `BodyRenderer` 实现。

### 1. 机器人系 (Bit-Bot) — 核心母机

- **视觉特性**：圆角几何体、金属光泽、机械天线。
- **材质表现**：Skin 层左上角 1px 白色高光，右下角 1px 半透明黑色遮罩，营造 3D 立体感。
- **关键动画**：`steps(1)` 机械式位移、屏幕像素流 (`0101`)、天线闪烁。
- **Agent 变色**：机身整体填充 Agent 色。

### 2. 植物系 (Quant-Flora / 量子仙人掌)

- **视觉特性**：自然绿色调、分形生长结构、顶部带刺或花蕾。
- **材质表现**：叶片带有亚光质感，顶部花蕾随 Token 消耗产生光晕。
- **关键动画**：周期性摆动（类似微风）、开花/落叶、随额度刷新产生的"复苏"效果。
- **Agent 变色**：刺的颜色或顶部花蕾颜色随 Agent 变化。
- **特殊机制**：Token 消耗相当于"浇水"。长期不使用（Sleeping）会变矮并布满像素灰尘。

### 3. 动物系 (Logic-Critters / 算力猫)

- **视觉特性**：毛绒感、大眼睛、拟人化表情。
- **材质表现**：利用 1px 的杂色点模拟皮毛质感，轮廓线更柔和。
- **关键动画**：眨眼、伸懒腰、耳朵抖动、快速奔跑（High TPS 时）。
- **Agent 变色**：猫的斑纹颜色随 Agent 变化（Claude 橙斑、Gemini 彩虹尾巴）。
- **配件挂载**：耳机挂在双耳间，喷气背包挂在腹部（像猫包）。

### 4. 史莱姆系 (Slime-Blobs / 幻色史莱姆)

- **视觉特性**：半透明、Q弹、形变极度夸张。
- **材质表现**：内部有流动的小气泡（或 Token 颗粒），整体具有折射感高光。
- **关键动画**：落地时的"压扁"与弹起、吞噬 Token 块、随状态产生的色彩波动。
- **Agent 变色**：整体透明度和内部填充物颜色随 Agent 变化。
- **特殊机制**：根据 I/O 比例改变形状。Output 多则变高瘦，Input 多则变扁平。

### 5. 水母系 (Neural-Jelly / 神经元水母)

- **视觉特性**：深海发光、半透明触须、悬浮脉冲。
- **材质表现**：头部（伞部）边缘带有动态发光（Glow）层，触须具有物理摆动模拟。
- **关键动画**：节奏性缩放（脉冲）、触须拖尾、思考时产生的高频放电特效。
- **Agent 变色**：整体发光颜色变化，触须末端带 Agent 特色粒子。
- **配件挂载**：配件以"寄生"方式贴在水母头部，或漂浮在周围。

## 四、核心机制：双轨状态机 (Dual-Track Logic)

### 轨道一：实时 Hook (瞬时反馈)

监听 CLI 工具的实时事件（通过本地 Hook 接收器 `127.0.0.1:15799/state` 接收事件），延迟 < 100ms：

| CLI 事件 | 触发状态 | 视觉反馈 |
|:---|:---|:---|
| `UserPromptSubmit`（用户提交请求） | **Thinking** | 高频微颤 + 屏幕扫描线 + 天线信号闪烁 |
| `SubagentStart`（子代理启动） | **Juggling** | 左右摇摆 + 掉落 Token 方块 |
| `MultiAgentActive`（多代理并行） | **Conducting** | 缩放波动 + 代码音符 |
| `StreamChunk`（流式响应中） | **Typing** | 机械敲击感 + `>>>` 滚动字符 |
| `Success`（请求成功） | **Success** | 360° 旋转 + 像素礼花 |
| `Error`（请求失败） | **Error** | 剧烈抖动 + 机身变红 + 黑烟 |
| `AuthRequired`（权限申请） | **Auth** | 无辜歪头 + 警告色呼吸 |
| `HighTPS`（Token 速率极高） | **Fast** | 拖影效果 + 尾部火焰 |
| `SearchStart`（搜索/检索） | **Searching** | 横向平移 + 雷达扫描线 |
| `Evolving`（配件解锁） | **Evolving** | 全屏闪烁 + 重组变形 |

### 轨道二：长期用量 (进化养成)

监听本地日志或 API 统计（约 30s 轮询周期）：

| 长期事件 | 触发状态/行为 | 视觉反馈 |
|:---|:---|:---|
| **Token 累积** | 解锁配件 | 头部/背部/随身插槽升级（芽 → 耳机 → 披风） |
| **额度刷新 (Resets)** | Sleeping → Idle | 宠物从打瞌睡苏醒，伸懒腰 |
| **额度充足** | Idle / 正常生长 | 植物长高、动物活跃 |
| **长期未使用** | Sleeping | 降至地面静止、呼吸灯变慢、冒 Zzz |

### 12 种核心状态映射 (The 12-State Machine)

| # | 状态 (Hook) | 表情 (Face) | 动作 (Motion) | 特效 (FX) |
|:---|:---|:---|:---|:---|
| 01 | **Idle** | 眨眼 `[ . . ]` | `steps(1)` 垂直悬浮 (-1px) | 无 |
| 02 | **Thinking** | 代码线 `[ - - ]` | 高频微颤 (0.1s) | 扫描线 + 天线红绿闪烁 |
| 03 | **Typing** | 滚动字符 `[ >>> ]` | 机械敲击 (0.15s) | 0/1 像素块弹出 |
| 04 | **Juggling** | 掉落方块 | 左右摇摆 (0.4s) | Token 方块掉落 |
| 05 | **Conducting** | 音符 `♪` | 缩放波动 (0.6s) | 代码音符 |
| 06 | **Success** | 大笑 `[ ^▽^ ]` | 360° 旋转 (0.6s, steps(4)) | 像素礼花 (金色下落) |
| 07 | **Error** | 眼睛变叉 `[ X X ]` | 剧烈抖动 (0.2s) | 机身变红 `#EA4335` |
| 08 | **Sleeping** | 闭线 `[ _ _ ]` | 呼吸变慢 (5s 周期) | Zzz 粒子上浮 |
| 09 | **Auth** | 锁头 `@` | 无辜歪头 (2s) | 透明度呼吸 (0.7-1.0) |
| 10 | **Fast** | 拖影眼 `[ o o ]` | 快速横移 (0.08s) | 尾部火焰残影 |
| 11 | **Searching** | 雷达眼 `[ ◉ ◉ ]` | 横向平移 (1s) | 扫描线 |
| 12 | **Evolving** | 进度条 `[ = = ]` | 闪烁重组 (0.3s) | 全屏透明度闪烁 |

## 五、进化与配件系统 (Growth & Accessories)

通过累计消耗的 Token 驱动宠物的视觉进化。

### 配件插槽 (Accessory Slots)

| 插槽 | Lv.1 (10k) | Lv.2 (100k) | Lv.3 (1M) | Lv.4 (10M) |
|:---|:---|:---|:---|:---|
| **头部 (Top)** | 萌芽 (Sprout) | 战术耳机 (Headset) | 天才光环 (Halo) | 复古天线 (Antenna) |
| **背部 (Back)** | 能量电池 (Battery) | 喷气背包 (Jetpack) | 赛博披风 (Cape) | — |
| **随身 (Side)** | 迷你僚机 (实时 TPS) | 代码云 (下代码雨) | — | — |

### 特殊解锁

- **里程碑**：10k → 100k → 1M → 10M Tokens。
- **动态权重**：如果某段时间内报错率极高，有概率解锁"破旧补丁"配件。

> **MVP 范围**：配件系统在 v1.0 中仅实现 Bit-Bot 的头部插槽（Lv.1 萌芽）。完整配件系统延至 v1.1。

## 六、macOS 面板 UI 规范 (Panel UI)

下拉面板（Popover）宽度固定为 **360px**，采用 macOS 原生毛玻璃质感 (`backdrop-filter: blur(20px)`)。

### 1. 顶部展示区 (110px)

- 居中展示当前活跃宠物的 **64×64 高清放大版**动画（`transform: scale(3)`，`image-rendering: pixelated`）。
- 下方动态显示当前 Agent 的性格标签（如：`CLAUDE / 热血程序员`）。

### 2. CLI 任务卡片

每张卡片对应一个已追踪的 CLI 工具：

- **双列进度条**：
  - 左侧：`Current`（当前会话用量）
  - 右侧：`Weekly`（周用量）
- **倒计时**：清晰显示额度刷新时间（如 `Resets in 1h 22m`）。
- **统计行**：今日 Token 用量 / 本周 Token 用量。

#### 额度剩余颜色阈值（最终定稿）

| 剩余比例 | 颜色 | 语义 |
|:---|:---|:---|
| **> 40%** | **绿色** (`#34A853`) | 充足，正常使用 |
| **10% - 40%** | **黄色** (`#FBBC05`) | 警告，注意消耗 |
| **< 10%** | **红色** (`#EA4335`) | 危险，即将耗尽 |

> **注意**：此规则替代 Ultimate-Spec v2.0 中的 `<60% / 60-85% / >85%` 三档阈值。百分比以**剩余额度 / 总额度**计算。

### 3. 统计底栏

- 显示全局 `今日用量`、`累计用量`。
- 右下角设置入口（⚙️）。

## 七、扩展性设计 (Scalability)

### 1. 资产替换协议

| 协议 | 职责 |
|:---|:---|
| `FaceProvider` | 提供 12 种状态的表情帧（SVG / Canvas 路径） |
| `BodyRenderer` | 提供机身渲染逻辑（支持 Agent 变色） |
| `MountPointMap` | 定义配件在不同身体上的挂载点坐标 |
| `PhysicsProfile` | 定义种族的物理特性（重力、弹性、透明度） |

### 2. 快速添加新种族的步骤

以"算力猫"为例：
1. 实现 `FaceProvider`：将 12 个机器人表情换成猫脸表情（眯眼、竖耳等）。
2. 实现 `BodyRenderer`：绘制猫的身体，保留 Agent 变色逻辑。
3. 配置 `MountPointMap`：指定耳机挂在猫耳处，喷气背包挂在猫背处。
4. **无需修改**：所有 Hook 逻辑、Token 统计、等级系统、面板 UI 自动生效。
